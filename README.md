# octopus-publishing-gradle-plugin

Gradle plugin that configures **JFrog Artifactory** publishing for a consumer
project. Distilled and modernised from
[`octopus-rm-gradle-plugin`](../octopus-rm-gradle-plugin), keeping the same
behaviour, environment variables and POM-customization helper, but stripping
out everything unrelated to JFrog publishing.

## Requirements

| | Build | Consumer |
| --- | --- | --- |
| JDK | 21 (Gradle toolchain) | **17+** |
| Gradle | 9.5.1 | **9.0+** |
| Bytecode target | 17 | n/a |
| JFrog Artifactory plugin | 6.0.4 | bundled by the plugin |

## What the plugin does

When applied to a project (typically the root), it walks `rootProject.allprojects { … }`
and, for each project:

1. Creates a default `mavenJava` `MavenPublication` from the `java`
   software component **only if** the project has `maven-publish` applied
   and a `java` component (e.g. via `java` / `java-library`).
   The plugin auto-applies `maven-publish` on the **root project only**
   (RM plugin parity); **subprojects must apply `maven-publish`
   themselves** (typically together with `java-library`).
2. Applies `com.jfrog.artifactory` to the root project **and every
   subproject** (parity with the legacy `setupRootPublishing` behaviour).
3. Reads connection / credential settings from the environment / project
   properties and configures the `artifactory { publish { ... } }` block:
   - context URL, repository key, deployer credentials,
   - `publishPom=true`, `publishArtifacts=true`, `publishBuildInfo=true`,
   - `publications('ALL_PUBLICATIONS')`.
4. Wires `publish` to depend on `artifactoryPublish` and disables the
   redundant `PublishToMavenRepository` tasks generated by `maven-publish`
   for the Artifactory-managed repository.

Projects without `maven-publish` (e.g. an aggregator root with no `java`
component) are left untouched: their `artifactoryPublish` task is marked
`skip = true` so the build does not fail.

## Applying the plugin

```kotlin
// build.gradle.kts (root or any publishing subproject)
plugins {
    `java-library`
    `maven-publish`
    id("org.octopusden.octopus-publishing")
}
```

```kotlin
// settings.gradle.kts
pluginManagement {
    plugins {
        id("org.octopusden.octopus-publishing") version settings.extra["octopus-publishing.version"] as String
    }
}
```

Declare the version once in `gradle.properties`:

```properties
octopus-publishing.version=<version>
```

## Configuration surface

### Extension DSL — `octopusPublishing { ... }`

```kotlin
octopusPublishing {
    devRepoKey.set("rnd-maven-dev-local")           // default
    releaseRepoKey.set("rnd-maven-release-local")   // default
}
```

POM metadata (`name`, `url`, `licenses`, `scm`, `developers`) is configured
the standard `maven-publish` way — the plugin does not customize POMs:

```kotlin
publishing {
    publications {
        create<MavenPublication>("mavenJava") {
            from(components["java"])
            pom {
                url.set("https://github.com/my-org/my-library")
                licenses {
                    license {
                        name.set("Apache-2.0")
                        url.set("https://www.apache.org/licenses/LICENSE-2.0.txt")
                    }
                }
            }
        }
    }
}
```

### Environment variables

| Name | Purpose |
| --- | --- |
| `ARTIFACTORY_URL` | Base Artifactory URL (e.g. `https://artifactory.example.com`). The plugin appends `/artifactory`. |
| `ARTIFACTORY_DEPLOYER_USERNAME` | Deploy user. |
| `ARTIFACTORY_DEPLOYER_PASSWORD` | Deploy password / API token. |

### Project properties

| Name | Purpose |
| --- | --- |
| `artifactoryUrl` | Fallback for `ARTIFACTORY_URL`. |
| `ARTIFACTORY_DEPLOYER_USERNAME` | Fallback for the env var of the same name. |
| `ARTIFACTORY_DEPLOYER_PASSWORD` | Fallback for the env var of the same name. |
| `publishToReleaseRepository=true` | Use `releaseRepoKey` instead of `devRepoKey`. Any other value (or absent) keeps the dev repo. |

### Default repository keys

| Setting | Default |
| --- | --- |
| `devRepoKey` | `rnd-maven-dev-local` |
| `releaseRepoKey` | `rnd-maven-release-local` |

### Behaviour notes

- When `ARTIFACTORY_URL` (and fallback property `artifactoryUrl`) is missing,
  the Artifactory configuration step is **skipped** (logged at INFO); the
  build does not fail. This matches the legacy plugin.
- The plugin walks `rootProject.allprojects` once on first apply and is
  idempotent: applying the id again (in a subproject) is a no-op guarded
  by an extra-property marker on the root project.
- Projects without `maven-publish` get `artifactoryPublish.skip = true`
  (the JFrog plugin's own skip mechanism), so multi-module builds with
  non-publishing modules succeed.

## Functional tests

The `ft/` subproject contains FTs that invoke a real Gradle build against
packaged test projects via `platformlib-process-local`:

| Scenario | Test class / method |
| --- | --- |
| Consumer-side `pom { … }` flows into the generated POM | `OctopusPublishingPluginFT.testPomCustomization` |
| Credential resolution (env vs project property) | `OctopusPublishingPluginFT.testCredentialResolution` |
| Skip when URL missing | `OctopusPublishingPluginFT.testArtifactorySkippedWhenUrlMissing` |
| Repo-key selection via `publishToReleaseRepository` | `OctopusPublishingPluginFT.testRepoKeySelection` |
| End-to-end upload to a real Artifactory | `OctopusPublishingPluginFT.testRealUpload` *(env-gated)* |
| Multi-module: root applies, every subproject publishes | `MultiModulePublishFT` |
| Root without publication: subprojects still publish, no failure | `RootOnlyNoPublishFT` |
| Root with `java-library` but no explicit `maven-publish`: plugin auto-applies and publishes root | `RootWithJavaNoMavenPublishFT` |
| Idempotency: plugin applied on root + subproject | `IdempotencyFT` |

Run the unit-style FTs (FT now runs as part of the standard `build`):

```bash
./gradlew clean build                # builds the plugin AND runs the FT suite
./gradlew :ft:test                   # FT only (depends on :publishToMavenLocal)
```

Run the real-upload FT (requires a real Artifactory):

```bash
export ARTIFACTORY_URL=https://artifactory.example.com
export ARTIFACTORY_DEPLOYER_USERNAME=...
export ARTIFACTORY_DEPLOYER_PASSWORD=...
export ARTIFACTORY_FT_REPO_KEY=rnd-maven-dev-local   # optional, defaults shown
./gradlew :ft:test --tests '*testRealUpload'
```

## Migration from `octopus-rm-gradle-plugin`

### Preserved (drop-in for the JFrog publishing surface)

| Item | Notes                                                                                                                                                         |
| --- |---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Env var `ARTIFACTORY_URL` | Same name & semantics.                                                                                                                                        |
| Env vars `ARTIFACTORY_DEPLOYER_USERNAME` / `ARTIFACTORY_DEPLOYER_PASSWORD` | Same names.                                                                                                                                                   |
| Project property `artifactoryUrl` | Fallback for the URL env var.                                                                                                                                 |
| Project property `publishToReleaseRepository` | Same flag, same effect.                                                                                                                                       |
| Default repo keys `rnd-maven-dev-local` / `rnd-maven-release-local` | Now also overridable via the extension.                                                                                                                       |
| Publication name `mavenJava` | Same publication name from `components.java`.                                                                                                                 |
| `publish.dependsOn(artifactoryPublish)` task wiring | Same.                                                                                                                                                         |
| `com.jfrog.artifactory` applied to root + every subproject | Same.                                                                                                                                                         |
| Multi-project behaviour | Root apply configures every subproject; projects without `maven-publish` get `artifactoryPublish.skip = true`. Same as `setupProjectPublishing` in RM plugin. |

### Changed

| Aspect | Before (`octopus-rm-gradle-plugin`) | Now (`octopus-publishing-gradle-plugin`) |
| --- | --- | --- |
| Build script DSL | Groovy (`build.gradle`) | **Kotlin** (`build.gradle.kts`) |
| Source layout | Monolithic `ReleaseManagementGradlePlugin.groovy` (~300 lines) | Slim Groovy bootstrap + focused Kotlin/Groovy classes (`PublishingConfigurer`, `ArtifactoryConfigurer`) |
| Repository keys | Hard-coded `rnd-maven-{dev,release}-local` | Configurable via `octopusPublishing { devRepoKey/releaseRepoKey }` (same defaults) |
| Plugin id | `org.octopusden.octopus-release-management` | `org.octopusden.octopus-publishing` |
| Extension name | (none — used Groovy metaClass tricks only) | `octopusPublishing { … }` |
| Group | `org.octopusden.octopus-release-management` | `org.octopusden.octopus.publishing` |
| Min Gradle | ~7.x | **9.0+** (built & tested against 9.5.1) |
| JDK target | 17 (build & runtime) | Build **21** (toolchain), bytecode/runtime **17** |
| JFrog Artifactory plugin | 4.x / 5.x | **6.0.4** |
| Kotlin (build) | n/a | **2.2.0** |
| Consumer version property | n/a | `octopus-publishing.version` (matches `octopus-oc-template.version` convention) |
| Plugin marker `displayName` / `description` | Hardcoded strings | Derived from `project.name` / `project.description` |
| Idempotency flag | `setupArtifactoryPublish` + `releaseManagementConfigurationState` | `setupOctopusPublishing` + `octopusPublishingConfigurationState` |
| FT layout | `ft/` Groovy DSL with `artifactory-manager` Kotlin subproject | `ft/` **Kotlin DSL** with a slimmer `artifactory-manager` subproject |
| FT coverage of JFrog publishing | None | `OctopusPublishingPluginFT` + `MultiModulePublishFT` + `RootOnlyNoPublishFT` + `IdempotencyFT` |

### Removed

The following responsibilities of `octopus-rm-gradle-plugin` are
**intentionally not** part of the new plugin:

| Removed                                                                                                                                                                                 | Reason |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------| --- |
| Dependency export (`ExportDependenciesToTeamcityTask`, `releaseManagement { releaseDependencies { … } }`, `ReleaseManagementDependenciesExtension`, `ReleaseDependenciesConfiguration`) | Not a publishing concern. |
| Components-registry integration & DTOs (`Component`, `VersionedComponent`, …)                                                                                                           | Not a publishing concern. |
| CycloneDX SBOM generation (`org.cyclonedx.bom` setup, `cyclonedx.skip` property)                                                                                                        | Out of scope. |
| Escrow mode (`m2_local` project property, offline-repository switching, `escrowBuild` flag)                                                                                             | Out of scope (instructions explicitly defer this). |
| Consumer-side **Sonatype Central** dual-publishing (`nexusPublishing { sonatype { … } }`)                                                                                               | The plugin provides **JFrog only** for consumers. |
| Consumer-side **GPG signing** (`signing { … }` gated on `nexus` property)                                                                                                               | Removed alongside Sonatype dual-publishing. |
| Deprecated alias plugin id `gradle-staging-plugin`                                                                                                                                      | Clean break under the new coordinate. |
| Legacy credential fallback `mavenUser` / `mavenPassword` (and `NEXUS_USER` / `NEXUS_PASSWORD` for the consumer)                                                                         | Only `ARTIFACTORY_DEPLOYER_USERNAME` / `_PASSWORD` are honoured. |
| `buildVersion` / version-override handling tied to RM plugin                                                                                                                            | Consumers set `project.version` themselves. |
| `setBuildVersion` + `1.0-SNAPSHOT` default fallback                                                                                                                                     | Consumers control their own versioning. |
| `setupArtifactoryPublish` ext flag (replaced by `setupOctopusPublishing`)                                                                                                               | Same idempotency, renamed for clarity. |
| `MavenPom.declareDependencies(Configuration…)` Groovy `metaClass` helper                                                                                                                | Unused by any known consumer; for `java-library` projects `from components.java` already produces a correct `<dependencies>` block. |

The plugin's own **release pipeline** still publishes to JFrog **and**
Sonatype Central with GPG signing (see `build.gradle.kts`); that is unrelated
to what the plugin offers its consumers.

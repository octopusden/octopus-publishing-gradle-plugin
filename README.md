# octopus-publishing-gradle-plugin

Gradle plugin that configures **JFrog Artifactory** publishing for a consumer
project. Distilled and modernized from octopus-rm-gradle-plugin, keeping the same
behavior, but stripping out everything unrelated to JFrog publishing.

## Requirements

|                          | Build                 | Consumer              |
|--------------------------|-----------------------|-----------------------|
| JDK                      | 21 (Gradle toolchain) | **17+**               |
| Gradle                   | 9.5.1                 | **9.0+**              |
| Bytecode target          | 17                    | n/a                   |
| JFrog Artifactory plugin | 6.0.4                 | bundled by the plugin |

## What the plugin does

Apply on the **root project**. For every project in the build, the plugin:

1. Applies `com.jfrog.artifactory`. Auto-applies `maven-publish` only on the root.
2. Creates a default `mavenJava` publication from the `java` component when
   both `maven-publish` and a `java` component are present.
3. Configures `artifactory { publish { ... } }` on the **root only** (context
   URL, repo key, deployer credentials, `publishBuildInfo=true`); subprojects
   inherit it. When `ARTIFACTORY_URL` is missing the step is logged and
   skipped, the build does not fail.
4. Wires `publish` to depend on `artifactoryPublish` and disables redundant
   `PublishToMavenRepository` tasks generated by `maven-publish`.
5. For projects without `maven-publish`, marks `artifactoryPublish.skip = true`
   so aggregate builds succeed.

Applying the id from a subproject is tolerated but not the supported pattern:
it resolves `project.rootProject` and bootstraps the same global setup.

## Applying the plugin

```kotlin
// build.gradle.kts (root project)
plugins {
    id("org.octopusden.octopus-publishing")
}
```

```kotlin
// settings.gradle.kts
pluginManagement {
    plugins {
        id("org.octopusden.octopus-publishing") version settings.extra["octopus-publishing.version"] as String
    }
}
```

Declare the version once in `gradle.properties`:

```properties
octopus-publishing.version=<version>
```

## Configuration surface

### Extension DSL — `octopusPublishing { ... }`

```kotlin
octopusPublishing {
    devRepoKey.set("rnd-maven-dev-local")           // default
    releaseRepoKey.set("rnd-maven-release-local")   // default
}
```

POM metadata (`name`, `url`, `licenses`, `scm`, `developers`) is configured
the standard `maven-publish` way — the plugin does not customize POMs:

```kotlin
publishing {
    publications {
        create<MavenPublication>("mavenJava") {
            from(components["java"])
            pom {
                url.set("https://github.com/my-org/my-library")
                licenses {
                    license {
                        name.set("Apache-2.0")
                        url.set("https://www.apache.org/licenses/LICENSE-2.0.txt")
                    }
                }
            }
        }
    }
}
```

### Environment variables

| Name | Purpose |
| --- | --- |
| `ARTIFACTORY_URL` | Base Artifactory URL (e.g. `https://artifactory.example.com`). The plugin appends `/artifactory`. |
| `ARTIFACTORY_DEPLOYER_USERNAME` | Deploy user. |
| `ARTIFACTORY_DEPLOYER_PASSWORD` | Deploy password / API token. |

### Project properties

| Name | Purpose |
| --- | --- |
| `artifactoryUrl` | Fallback for `ARTIFACTORY_URL`. |
| `ARTIFACTORY_DEPLOYER_USERNAME` | Fallback for the env var of the same name. |
| `ARTIFACTORY_DEPLOYER_PASSWORD` | Fallback for the env var of the same name. |
| `publishToReleaseRepository=true` | Use `releaseRepoKey` instead of `devRepoKey`. Any other value (or absent) keeps the dev repo. |

### Default repository keys

| Setting          | Default                   |
|------------------|---------------------------|
| `devRepoKey`     | `rnd-maven-dev-local`     |
| `releaseRepoKey` | `rnd-maven-release-local` |

## Functional tests

The `ft/` subproject contains FTs that invoke a real Gradle build against
packaged test projects via `platformlib-process-local`.

Run the unit-style FTs (FT now runs as part of the standard `build`):

```bash
./gradlew clean build                # builds the plugin AND runs the FT suite
./gradlew :ft:test                   # FT only (depends on :publishToMavenLocal)
```
